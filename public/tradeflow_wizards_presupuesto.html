Presupuestos por Pasos
Wizard por Oficio

Algunos oficios no pueden presupuestarse con una simple lista de partidas. Mudanzas, redes informáticas, reformas integrales… necesitan un flujo guiado de preguntas porque cada respuesta cambia radicalmente el presupuesto. Esta arquitectura define qué oficios lo necesitan, cuántos pasos tiene cada uno y cómo se almacena en BD para que la IA genere el presupuesto final.

¿Qué oficios necesitan Wizard?

Matriz de decisión
Criterio de decisión: Un oficio necesita wizard cuando el precio depende de variables que el instalador no puede capturar por voz de forma fiable — dimensiones, cantidades estructuradas, condiciones especiales que si se omiten generan un presupuesto inútil o muy alejado de la realidad.
Oficio ¿Wizard? Motivo clave Pasos Alternativa si no
Mudanzas Sí M³, plantas, ascensor, guardamuebles, licencia, distancia — cada variable cambia el precio drásticamente 7 pasos —
Redes Informáticas / IT Sí Puntos de red, m cable, rack, servidores, tipo de cableado, certificación — sin esto el presupuesto es inválido 6 pasos —
Reformas Integrales Sí Estancias, m², tipo de material, vaciado, proyecto, licencia — demasiadas variables para voz 8 pasos —

04 —

Motor IA — Del Wizard al Presupuesto

Cómo funciona: El wizard captura datos estructurados → el backend aplica las reglas de wizard_partida_rules y genera las partidas automáticamente sin IA → la IA solo entra para completar lo que las reglas no cubren, mejorar las descripciones y detectar partidas olvidadas.
fase 01
Usuario completa wizard paso a paso
fase 02
Backend aplica wizard_partida_rules → genera partidas
fase 03
IA revisa, mejora descripciones y detecta gaps
fase 04
Instalador revisa y ajusta precios del catálogo
fase 05
Presupuesto generado → PDF → WhatsApp/email
system_prompt_wizard — extracto para Mudanzas # Recibes las respuestas del wizard de Mudanzas

# → añadir partida recargo_sin_ascensor

- materiales NO incluidos salvo indicación expresa
- distancia > 300km → partida alojamiento técnicos
- grua_exterior = true → partida independiente
- m3 > 60 → sugerir 2 camiones
